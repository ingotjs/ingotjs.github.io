Why Prospect?

Route-based E2E coverage tracking — type-safe, explicit, validated on every run.

Declarative Coverage Maps

defineE2ECoverage() maps routes to their interactive elements. Your coverage spec is code — type-safe, version-controlled, IDE-refactorable.

Explicit Gap Tracking

test: null marks coverage gaps. Visible in code review, impossible to ignore. No more hoping you tested everything.

Validation on Every Run

setup() plugs into Playwright's globalSetup. Missing test files, duplicate IDs, orphaned selectors — caught automatically before tests execute.

Shared Component Groups

Extract headers, footers, and menus as variables. Spread them into routes. One source of truth — change a selector in one place, every route updates.

Coverage as code

Define what interactions exist on each route. Map them to test files — or mark them null to track the gap explicitly. Prospect validates the whole picture on every test run.

coverage.ts
const e2e = defineE2ECoverage({
  testId,
  routes: {
    "/auth/sign-in": {
      interactions: {
        [testId.signin.submit]: [
          {
            context: "valid",
            expected: "navigate to /dashboard",
            test: "sign-in.e2e.ts",
          },
          {
            context: "invalid",
            expected: "shows error",
            test: "sign-in.e2e.ts",
          },
        ],
      },
    },
    "/dashboard": {
      interactions: {
        [testId.dashboard.create]: [
          {
            expected: "opens modal",
            test: null, // ← gap
          },
        ],
      },
    },
  },
})
